## 1. 产品愿景

**Flux OS** 是一个 AI-First 的智能金融终端。

核心理念：**Signal over Noise** — 将海量金融噪音转化为高信噪比的决策信号。

| 传统方案 | Flux OS |
|---------|---------|
| 5 个网页分散操作 | 一屏聚合所有决策信息 |
| 信息爆炸，用户自己过滤 | AI 提炼核心观点和信号 |
| AI 是附加插件 | AI 是核心交互逻辑，驱动数据获取和分析 |
| 机构级分析门槛高 | 个人投资者获得机构分析师级别的辅助 |

**产品哲学**:
- **AI First**: AI 是产品的中枢神经，从数据检索到分析推理无处不在
- **极简主义**: 只展示对决策有用的数据，隐藏噪音
- **私有化**: 为内部小团队设计，数据不离开自有基础设施

> 详见 [docs/v0.01/why flux.md](./v0.01/why%20flux.md) — 用户体验旅程地图

### v0.03 — 功能跑通 (已完成)

**目标**: 将核心占位功能升级为真实可用，然后尽快进入 AI 能力升级线。

| Phase | 功能 | 状态 | 优先级 |
|-------|------|------|-------|
| P1 | AI 问答 + ContextInput | ✅ 已完成 | — |
| P2 | Dashboard 数据接入 (持仓/盈亏/风险) | ✅ 已完成 | — |
| ~~P4~~ | ~~新闻情感分析~~ | 推迟至 v0.05+ | — |

**决策 (2026-02)**: 原 P3 (情感分析) 推迟，原 P4 (财报) 曾提前为新 P3，后于 2026-04 从产品中移除：
- **情感分析 → 推迟**: 当前新闻源 (RSS 聚合 + Finnhub 兜底) 时效滞后、标题信息量不足，AI 情感标签在低质量数据上 ROI 不高。v0.04 Tool Calling 落地后，AI 可实时搜索+分析新闻，比静态标签更有价值。
- **财报分析 → 已移除**: Detail 财报 tab 与相关 transcript / earnings API 已在 2026-04 下线，当前产品不再包含该功能。

v0.03 全部 Phase 已完成，下一步进入 v0.04。

### v0.04 — Agentic AI (下一步)

**目标**: AI 架构跃迁——从"预收集上下文 + 无状态单轮"升级为"按需获取数据 + 多轮对话 + 会话持久化"。

**决策 (2026-02)**: 合并原 v0.04 (多轮对话) 和原 v0.05 (Tool Calling)：
- **分开做的问题**: v0.04 手写 streaming + 消息管理 → v0.05 全部推翻换 `useChat`，重复工作
- **合并的好处**: 直接用 AI SDK `useChat`，一次完成前端状态管理迁移；`collectChatContext` 预收集模式直接废弃，AI 按需调 tool 获取数据

核心交付：

| 能力 | 说明 |
|------|------|
| **Tool Calling** | AI 自主调用工具查行情、搜新闻、算指标，按需获取数据 |
| **多轮对话** | 对话历史持久化到 DB，支持上下文连续追问 |
| **会话管理** | 新建 / 切换 / 删除 / 重命名会话，自动恢复最近会话 |
| **Tool Call UI** | 对话流中显示工具调用 loading 态 (数据卡片后续迭代) |
| **前端迁移** | 从手写 fetch + ReadableStream 迁移到 AI SDK `useChat` |

**附带功能** (依赖 Tool Calling 基础设施):
- 新闻情感分析 — AI 实时搜索+分析，替代静态标签方案

> 技术设计将在 `docs/v0.04/design.md` 中详述 (合并原 v0.04 + v0.05 草案)

### v0.07 — Detail 页面 UI/UX 打磨

**目标**: 修复 Detail 页面的交互问题和视觉体验短板，提升信息密度和可读性。

核心交付：

| 优先级 | 问题 | 说明 |
|--------|------|------|
| **P0** | 涨跌幅与图表周期不同步 | 百分比应随周期切换动态计算，而非固定 Quote 值 |
| **P0** | AI 面板空间不足 | 左右面板改为可拖拽分割线，用户自由调整比例，持久化到 localStorage |
| **P1** | 缺少公司名称 | 头部仅显示 ticker，需展示 StockInfo.name |
| **P1** | AI 研报纯文本墙 | 需结构化渲染（核心观点/风险提示卡片），对齐 PRD 设计 |
| **P1** | 图表无 Y 轴刻度 | 添加 2-3 个浅色刻度线辅助读数 |
| **P2** | ContextInput 跨 Tab 行为无提示 | 研报/财报 Tab 下输入会跳转问答，需视觉暗示 |
| **P2** | QuarterSwitcher 原生 select | 替换为自定义下拉，统一深色主题 |
| **P3** | 新闻列表缺乏视觉区分 | 增加重要性指标或时间分组 |
| **P3** | 指标卡片空状态 | 股息率 `--` 需说明文案（如"暂不派息"） |

> 详见 [docs/v0.07/detail-uiux-polish.md](./v0.07/detail-uiux-polish.md)
